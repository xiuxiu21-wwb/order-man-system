## 🚀 准备工作

### 1.1 购买阿里云服务器
1. 访问：https://www.aliyun.com/product/ecs
2. 选择配置（推荐）：
   - **操作系统**：Ubuntu 22.04 LTS 或 CentOS 7
   - **实例规格**：2核2GB（个人项目足够）
   - **带宽**：3Mbps起
   - **系统盘**：40GB SSD
3. 购买完成后，记录：
   - 公网IP地址
   - 实例用户名（通常是 root）
   - 登录密码或SSH密钥

### 1.2 配置安全组
登录阿里云控制台 → 云服务器ECS → 安全组 → 配置规则：

**入方向规则**：
| 协议端口 | 授权对象 | 说明 |
|---------|---------|------|
| TCP:22 | 0.0.0.0/0 | SSH远程登录 |
| TCP:80 | 0.0.0.0/0 | HTTP访问 |
| TCP:443 | 0.0.0.0/0 | HTTPS访问 |
| TCP:8000 | 0.0.0.0/0 | 后端API（可选） |

## 📦 后端部署

### 4.1 上传项目代码
方式一：使用 Git 克隆（推荐）
```bash
# 先在 GitHub 上传代码，然后克隆
cd /opt
git clone https://github.com/你的用户名/order-man-system.git
cd order-man-system
```

方式二：使用 SCP 上传
```bash
# 在本地 PowerShell 执行
scp -r "c:\Users\20924\Desktop\02_项目代码\order man\ord man" root@你的公网IP:/opt/
```

### 4.2 创建虚拟环境
```bash
cd /opt/ord-man
python3.11 -m venv venv
source venv/bin/activate
```

### 4.3 安装依赖
```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 4.4 初始化数据库
```bash
python init_db.py
python create_admin.py
```

### 4.5 使用 Gunicorn + Uvicorn 部署
```bash
pip install gunicorn uvicorn-workers
```

创建启动脚本：
```bash
cat > /opt/ord-man/start_backend.sh << 'EOF'
#!/bin/bash
cd /opt/ord-man
source venv/bin/activate
gunicorn app.main:app \
  --workers 4 \
  --worker-class uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8000 \
  --access-logfile /var/log/backend-access.log \
  --error-logfile /var/log/backend-error.log
EOF

chmod +x /opt/ord-man/start_backend.sh
```

### 4.6 配置 Systemd 服务
```bash
cat > /etc/systemd/system/order-backend.service << 'EOF'
[Unit]
Description=Order Management Backend
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/ord-man
Environment="PATH=/opt/ord-man/venv/bin"
ExecStart=/opt/ord-man/venv/bin/gunicorn app.main:app --workers 4 --worker-class uvicorn.workers.UvicornWorker --bind 127.0.0.1:8000
Restart=always

[Install]
WantedBy=multi-user.target
EOF

# 启动服务
systemctl daemon-reload
systemctl start order-backend
systemctl enable order-backend
systemctl status order-backend
```

## 🎨 前端部署

### 5.1 修改前端 API 地址
编辑 `frontend/src` 中的文件，将 API 地址改为：
```javascript
// 开发环境
const API_BASE = 'http://你的公网IP/api';

// 或使用域名
const API_BASE = 'https://api.wwblsc.top/api';
```

### 5.2 构建前端
```bash
cd /opt/ord-man/frontend
npm install
npm run build
```

### 5.3 配置 Nginx
```bash
cat > /etc/nginx/sites-available/order-man << 'EOF'
server {
    listen 80;
    server_name 你的公网IP www.wwblsc.top;

    # 前端静态文件
    location / {
        root /opt/ord-man/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket支持
    location /ws {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
EOF

# 启用配置
ln -s /etc/nginx/sites-available/order-man /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 重启Nginx
systemctl restart nginx
```

## 🔐 域名与SSL配置

### 6.1 配置DNS解析
在阿里云控制台 → 域名 → 解析：
| 主机记录 | 记录类型 | 记录值 |
|---------|---------|-------|
| @ | A | 你的公网IP |
| www | A | 你的公网IP |
| api | A | 你的公网IP |

### 6.2 申请免费SSL证书（Let's Encrypt）
```bash
# 安装Certbot
apt install -y certbot python3-certbot-nginx

# 申请证书（自动配置Nginx）
certbot --nginx -d wwblsc.top -d www.wwblsc.top -d api.wwblsc.top

# 自动续期
certbot renew --dry-run
```

### 6.3 更新Nginx HTTPS配置
Certbot会自动更新配置，确保HTTP重定向到HTTPS。

```bash
# 检查后端
systemctl status order-backend

# 检查Nginx
systemctl status nginx

# 查看日志
journalctl -u order-backend -f
tail -f /var/log/nginx/access.log
```

### 8.2 防火墙配置
```bash
# Ubuntu/Debian (UFW)
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw enable

# CentOS (firewalld)
firewall-cmd --permanent --add-port=22/tcp
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --permanent --add-port=443/tcp
firewall-cmd --reload
```

## ⚠️ 注意事项

1. **定期备份数据库**：SQLite数据库文件在项目目录中
2. **监控服务器资源**：使用 `htop` 或 `top` 查看
3. **安全更新**：定期运行 `apt update && apt upgrade`
4. **日志轮转**：配置logrotate防止日志文件过大
5. **防火墙**：只开放必要的端口
