# 阿里云安全组配置指南

## 什么是安全组？
安全组是阿里云的虚拟防火墙，用于控制进出 ECS 实例的网络流量。

## 配置步骤

### 步骤 1：进入安全组配置页面
1. 登录阿里云控制台
2. 进入 ECS 实例详情页
3. 点击左侧菜单 "网络与安全组" -> "安全组配置"

### 步骤 2：添加入方向规则
1. 点击 "配置规则" 按钮
2. 切换到 "入方向" 标签
3. 点击 "手动添加" 按钮

### 步骤 3：添加以下规则

#### 规则 1：后端 API 端口
- 优先级：1
- 策略：允许
- 协议类型：TCP
- 端口范围：`8000/8000`
- 授权对象：`0.0.0.0/0`
- 描述：后端 API 服务

#### 规则 2：HTTP 端口
- 优先级：1
- 策略：允许
- 协议类型：TCP
- 端口范围：`80/80`
- 授权对象：`0.0.0.0/0`
- 描述：HTTP Web 服务

#### 规则 3：HTTPS 端口
- 优先级：1
- 策略：允许
- 协议类型：TCP
- 端口范围：`443/443`
- 授权对象：`0.0.0.0/0`
- 描述：HTTPS Web 服务

#### 规则 4：前端开发端口（可选）
- 优先级：1
- 策略：允许
- 协议类型：TCP
- 端口范围：`3000/3000`
- 授权对象：`0.0.0.0/0`
- 描述：前端开发服务器

#### 规则 5：远程桌面（如果还没有）
- 优先级：1
- 策略：允许
- 协议类型：TCP
- 端口范围：`3389/3389`
- 授权对象：`0.0.0.0/0`
- 描述：远程桌面连接

### 步骤 4：保存配置
点击 "保存" 按钮

## 验证配置

### 方法 1：使用浏览器测试
在浏览器中访问：
```
http://你的服务器公网 IP:8000/api/users/test
```

应该返回：
```json
{
  "message": "API is working",
  "register_url": "/api/users/register (POST)"
}
```

### 方法 2：使用 telnet 测试
在本地电脑打开 PowerShell：
```powershell
telnet 你的服务器公网 IP 8000
```

如果连接成功，说明端口已开放。

## 常见问题

### Q1: 配置后仍然无法访问？
**A**: 可能还需要在 Windows 防火墙中开放端口：
```powershell
# 在服务器上运行（管理员权限）
netsh advfirewall firewall add rule name="OrderMan Backend" dir=in action=allow protocol=TCP localport=8000
netsh advfirewall firewall add rule name="HTTP" dir=in action=allow protocol=TCP localport=80
netsh advfirewall firewall add rule name="HTTPS" dir=in action=allow protocol=TCP localport=443
```

### Q2: 授权对象 0.0.0.0/0 是什么意思？
**A**: 表示允许所有 IP 地址访问。生产环境建议设置为特定的 IP 段。

### Q3: 优先级是什么意思？
**A**: 数字越小优先级越高。相同优先级的规则，"拒绝" 优先于 "允许"。

### Q4: 需要重启服务器吗？
**A**: 不需要，安全组规则立即生效。

## 安全建议

1. **最小权限原则**：只开放必要的端口
2. **定期审查**：定期检查安全组规则，删除不需要的规则
3. **限制 IP**：如果可能，限制访问 IP 范围
4. **使用 HTTPS**：生产环境务必使用 HTTPS

## 端口说明

| 端口 | 用途 | 是否必须 |
|------|------|----------|
| 8000 | 后端 API | ✅ 必须 |
| 80 | HTTP | ✅ 必须（测试用） |
| 443 | HTTPS | ✅ 必须（生产用） |
| 3000 | 前端开发 | ❌ 可选 |
| 3389 | 远程桌面 | ⚠️ 建议限制 IP |
| 22 | SSH | ❌ Windows 不需要 |
