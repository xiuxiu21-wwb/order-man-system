# 文件上传到服务器的方法

## 方法一：使用阿里云 Workbench 上传（推荐）

### 步骤：
1. 在阿里云控制台，点击你的 ECS 实例
2. 点击 "远程连接" 按钮
3. 选择 "Workbench" 方式登录
4. 登录后，点击右上角的 "文件传输" 图标
5. 直接将本地文件拖拽到右侧窗口
6. 或者点击 "上传文件" 按钮选择文件

### 优点：
- ✅ 简单直接
- ✅ 支持大文件
- ✅ 不需要额外软件

---

## 方法二：使用远程桌面的本地磁盘映射

### 步骤：
1. 在本地电脑打开 "远程桌面连接"（mstsc）
2. 点击 "显示选项"
3. 切换到 "本地资源" 标签
4. 点击 "详细信息" 按钮
5. 勾选 "驱动器"（或展开选择特定驱动器）
6. 点击 "确定"，然后连接服务器

### 连接后：
1. 在服务器上打开 "此电脑"
2. 你会看到本地磁盘映射为网络驱动器
3. 直接复制文件即可

### 如果还是不行：
在远程桌面中按 `Ctrl + Alt + End` 可以调出任务管理器，然后：
1. 文件 -> 运行新任务
2. 输入 `explorer.exe`
3. 在文件资源管理器中查找本地磁盘

## 方法三：使用 FTP 工具（FileZilla）

### 步骤：
1. 在本地电脑下载 FileZilla：https://filezilla-project.org/
2. 安装并打开 FileZilla
3. 配置服务器连接：
   - 主机：你的服务器公网 IP
   - 用户名：Administrator
   - 密码：你的服务器密码
   - 端口：22（需要先开启 SSH）

### 开启 SSH（如果需要）：
1. 在阿里云控制台，进入 ECS 实例
2. 点击 "远程连接"
3. 使用 Workbench 登录
4. 运行以下命令开启 SSH：
```powershell
# 在 PowerShell 中运行
Set-Service -Name sshd -StartupType 'Automatic'
Start-Service sshd
```

## 方法四：使用 Git 上传（如果你有 GitHub/Gitee）

### 步骤：
1. 在本地电脑上传代码到代码仓库：
```bash
cd "c:\Users\20924\Desktop\order man2\ord man"
git init
git add .
git commit -m "部署版本"
git remote add origin 你的仓库地址
git push -u origin master
```

2. 在服务器上克隆代码：
```powershell
cd C:\
git clone 你的仓库地址 ord-man
```

## 方法五：使用压缩包上传（最简单）

### 步骤：
1. 在本地电脑压缩 `ord man` 文件夹为 ZIP 文件
2. 使用阿里云 Workbench 的文件上传功能
3. 上传 ZIP 文件到服务器
4. 在服务器上解压：
   - 右键点击 ZIP 文件
   - 选择 "全部解压缩"
   - 解压到 `C:\ord man`

---

## 方法六：使用 PowerShell 远程传输

### 在本地电脑 PowerShell 中运行：
```powershell
# 建立远程会话
$session = New-PSSession -ComputerName 你的服务器IP -Credential (Get-Credential)

# 复制文件
Copy-Item -Path "C:\Users\20924\Desktop\order man2\ord man" -Destination "C:\" -ToSession $session -Recurse

# 关闭会话
Remove-PSSession $session
```

## 推荐方案

**最简单的方法组合：**
1. 使用 **方法五（压缩包上传）** + **阿里云 Workbench**

## 常见问题

### Q: Workbench 在哪里？
A: 阿里云控制台 -> ECS 实例 -> 远程连接 -> Workbench

### Q: 上传速度慢怎么办？
A: 
- 压缩文件后再上传
- 删除不必要的文件（node_modules、__pycache__等）
- 使用更快的网络

### Q: 上传后文件在哪里？
A: 默认在你上传时选择的位置，建议在 `C:\ord man`
