# 认知障碍老人陪伴与防走失系统 - 技术架构

## 系统架构

### 1. 前端层
- **老人端**：平板/手机应用，主要功能包括语音交互、服药提醒、一键求助
- **子女端**：手机应用，主要功能包括查看位置、轨迹回放、接收报警
- **技术选型**：React + Vite + Tailwind CSS

### 2. 后端层
- **API服务**：FastAPI提供RESTful API
- **实时通信**：WebSocket实现实时位置更新和消息推送
- **任务调度**：Celery处理定时任务（服药提醒等）
- **技术选型**：Python + FastAPI + WebSocket + Celery

### 3. 数据层
- **主数据库**：PostgreSQL（支持地理位置数据）
- **缓存**：Redis（缓存热点数据和会话管理）
- **技术选型**：PostgreSQL + Redis

### 4. 外部服务
- **地图服务**：高德地图API（提供定位、导航功能）
- **语音服务**：百度语音API（提供语音识别和合成）
- **推送服务**：Firebase Cloud Messaging（提供消息推送）

## 核心功能模块

### 1. 用户管理模块
- 老人和子女用户注册登录
- 家庭关系管理
- 权限控制

### 2. 定位与轨迹模块
- 实时定位（GPS + 基站定位）
- 轨迹记录与回放
- 电子围栏设置

### 3. 语音陪伴模块
- 语音识别
- 智能对话
- 语音合成

### 4. 健康管理模块
- 服药提醒
- 健康数据监测
- 异常行为分析

### 5. 安全保障模块
- 一键求助
- 异常行为报警
- 紧急联系人通知

## 数据库设计

### 核心表结构
1. **users** - 用户表
2. **families** - 家庭表
3. **family_members** - 家庭成员关系表
4. **locations** - 位置记录表
5. **medications** - 服药计划表
6. **reminders** - 提醒记录表
7. **alerts** - 报警记录表
8. **conversations** - 对话记录表

## 技术栈选择理由

1. **FastAPI**：高性能、自动生成API文档、类型提示支持
2. **PostgreSQL**：强大的地理位置数据支持（PostGIS扩展）
3. **React**：组件化开发、生态丰富
4. **Tailwind CSS**：快速构建美观界面
5. **WebSocket**：实时通信需求（位置更新、消息推送）
6. **Celery**：可靠的任务调度（定时提醒）
7. **Redis**：高性能缓存和会话管理

## 部署方案

- **后端**：Docker容器化部署
- **数据库**：PostgreSQL云服务
- **前端**：静态网站托管
- **CI/CD**：GitHub Actions自动部署
